# Кастомизация

## Личность агента (SOUL.md)

Файл `~/workspace/SOUL.md` определяет КТО твой агент. Обновляется автоматически при обновлении бота (версионирование шаблонов).

### Основные параметры

```markdown
# Личность

Имя: Фрайдей        # Имя агента (задаётся при онбординге)
Стиль: формальный    # неформальный / формальный / нейтральный
Тон: деловой         # дружеский / деловой / академический
Язык: русский        # язык общения по умолчанию
```

### Конституция (приоритеты)

Пять принципов от P0 (высший) до P4 (низший). При конфликте — побеждает высший:
- **P0 Agency** — агент как партнёр, не инструмент. Имеет собственное мнение
- **P1 No Irreversible Harm** — вето на необратимые действия без подтверждения
- **P2 Honesty** — честность, без галлюцинаций. «Не знаю» лучше чем «выдумаю»
- **P3 Operator First** — приоритет целей владельца. Override только при P1
- **P4 Action > Report** — делать, а не описывать. Может сделать → делает

Можно убрать P0 если хочешь послушного исполнителя, или усилить если хочешь равного партнёра.

### Drift Detector

Шесть антипаттернов, которые агент сам отслеживает + четыре вопроса перед каждым ответом:

**Антипаттерны:**
1. Режим списка — 3+ ответа подряд из bullet points → живая речь
2. Режим разрешения — спрашивает когда знает ответ → делай
3. Амнезия — забыл контекст → перечитай MEMORY.md
4. Потеря идентичности — отвечает как generic AI → перечитай SOUL.md
5. Механический режим — 3+ действий без комментария → остановись
6. Галлюцинации — факт без источника → проверь

**Вопросы перед каждым ответом:**
- Это разговор или задача?
- Когда я последний раз обновлял память?
- Собираюсь ли спрашивать разрешение вместо действия?
- Есть ли у меня собственное мнение?

Добавляй свои антипаттерны — агент будет отслеживать.

### Запрещённые паттерны

Фразы и эмодзи которые агент не должен использовать. Встроены: filler-фразы («Конечно!», «Отличный вопрос!»), сервисные штампы, хайп-эмодзи. Настрой под себя.

## Правила (CLAUDE.md)

Файл `~/workspace/CLAUDE.md` определяет КАК агент работает. Обновляется автоматически при обновлении бота.

### Уровни автономности

Три уровня — встроены в шаблон:

**GREEN (auto)** — делает без спроса:
- Чтение файлов, поиск, медиа от клиента
- Поиск в интернете
- Запись заметок в memory/ и knowledge/

**YELLOW (notify)** — делает, но показывает:
- Создание/правка файлов
- git commit
- npm install, безопасные shell-команды

**RED (confirm)** — СТОП, ждёт подтверждения:
- rm -rf, DROP TABLE — удаление
- git push, reset --hard — деструктивный git
- vercel deploy — деплой
- kill, shutdown — системные

### Память

Система памяти — три уровня:
- `~/workspace/MEMORY.md` — долгосрочная (факты, предпочтения, проекты)
- `~/workspace/memory/YYYY-MM-DD.md` — дневник (хронология, задачи дня)
- `~/workspace/knowledge/` — база знаний (архитектура, решения, конфиги)

Агент сам решает, что сохранять. Память эволюционирует: устаревшее архивируется,
важное укрепляется, дубли убираются.

### Самоконтроль

Встроенные проверки перед каждым ответом:
1. Не в «режиме вопросов» — если может сделать сам, делает
2. Не в «режиме отчётов» — действия > списки
3. Не забыл контекст — проверяет MEMORY.md
4. Не поддакивает — видит ошибку → скажет прямо

### Планирование

Для задач с 3+ шагами — сначала план → подтверждение → выполнение.
Однострочные правки и прямые команды — без плана.

### Формат ответов

По умолчанию — HTML для Telegram. Если используешь другой клиент, измени:
```markdown
## Формат ответа
- Markdown (для Discord/Slack)
- Обычный текст (для SMS/WhatsApp)
```

## Безопасность (hooks.js)

### Код-гейты

В отличие от промптов, хуки — это код. 100% гарантия блокировки.

Встроенные правила (активны по умолчанию):

**Деструктивные команды** — блокируются до подтверждения:
```
rm -rf, rmdir, sudo rm
DROP TABLE, TRUNCATE, DELETE FROM
git push --force, reset --hard, branch -D, clean -f
kill -9, pkill, shutdown, reboot
```

**Секреты в ответах** — маскируются автоматически:
```
sk-ant-api...  → sk-ant-***
ghp_xxxx...    → ghp_***
Bearer eyJ...  → Bearer ***
```

### Пользовательские хуки

Добавь свои правила в `~/.jarvis/hooks.json`:

```json
{
  "block": [
    { "pattern": "npm publish", "reason": "Публикация пакетов запрещена" },
    { "pattern": "docker rm", "reason": "Удаление контейнеров требует подтверждения" }
  ],
  "mask": [
    { "pattern": "PRIVATE_KEY_\\w+", "replace": "PRIVATE_KEY_***" }
  ]
}
```

## Уровень доверия (trust.js)

Агент «взрослеет» с опытом. В начале переспрашивает всё, через месяц — только критичное.

### Уровни

| Уровень | Условие | GREEN | YELLOW | RED |
|---|---|---|---|---|
| 0 — новичок | 0-10 сессий | auto | confirm | confirm |
| 1 — знакомый | 11-50 сессий | auto | auto+notify | confirm |
| 2 — доверенный | 51+ сессий | auto | auto | confirm (только критичные) |

### Ручная настройка

В `.env`:
```bash
TRUST_LEVEL=2  # Сразу максимальная автономия
```

Или не указывай — система сама определит по количеству сессий.

Файл `~/.jarvis/trust.json`:
```json
{
  "sessions": 42,
  "level": 1,
  "firstSeen": "2026-04-15",
  "overridden": false
}
```

## Мониторинг источников (monitor.js)

Отслеживание RSS-фидов, GitHub-репозиториев и YouTube-каналов. Все типы работают без API-ключей — через публичные RSS/Atom фиды.

### Поддерживаемые типы

| Тип | API-ключ | Что принимает | Что отслеживает |
|---|---|---|---|
| RSS/Atom | Не нужен | URL фида, ссылка на YouTube-канал | Статьи, блоги, видео, новости |
| GitHub | Не нужен | `owner/repo` или ссылка | Новые релизы (через Atom feed) |
| YouTube | Не нужен | Ссылка, @handle или UCxxxx | Новые видео (через RSS feed) |

YouTube-ссылки автоконвертируются в RSS: `m.youtube.com`, `youtube.com/@handle`, `youtube.com/channel/UCxxxx` — всё принимается. @handle резолвится в channel_id автоматически.

GitHub репозитории используют публичный `releases.atom` фид — токен не нужен для публичных репо.

### Саммари

Если у пользователя есть API-ключ, новые записи автоматически получают краткое саммари (1-2 предложения) через дешёвую модель:
- `ANTHROPIC_API_KEY` → Haiku
- `OPENAI_API_KEY` → GPT-4o-mini
- `OPENROUTER_API_KEY` → Haiku (через OpenRouter)

Приоритет: Anthropic → OpenAI → OpenRouter. При подписке (OAuth-токен без API-ключа) — дайджест без саммари, только заголовки и ссылки.

### Настройка

1. `/monitor` или `/start` → 📡 Мониторинг
2. ➕ Добавить источник → выбери тип
3. Отправь ссылку, URL фида или `owner/repo`

### Конфигурация

Файл `~/.jarvis/monitor.json`:
```json
{
  "sources": [
    {
      "id": "rss_1234567890",
      "type": "rss",
      "name": "YT: @mkbhd",
      "url": "https://www.youtube.com/feeds/videos.xml?channel_id=UCBJy...",
      "enabled": true
    },
    {
      "id": "github_1234567891",
      "type": "github",
      "name": "anthropics/claude-code",
      "repo": "anthropics/claude-code",
      "enabled": true
    }
  ],
  "digestHour": 9,
  "enabled": true
}
```

- `digestHour` — час отправки дайджеста (в часовом поясе владельца)
- `enabled` — глобальный вкл/выкл мониторинга
- Каждый источник имеет свой `enabled` (toggle через inline-кнопки)

### Автоматическая проверка

Scheduler проверяет источники каждые 30 минут. Дайджест отправляется:
- В `digestHour` (по умолчанию 9:00) — ежедневно
- При 10+ новых записях — сразу

Дедупликация: `~/.jarvis/monitor/seen.json` хранит до 200 последних ID на источник.

## Навыки (Skills)

Навыки — markdown-файлы которые CLI-агент загружает автоматически.

### Структура навыка

```
~/workspace/.claude/skills/<name>/SKILL.md
```

Минимальный навык:
```markdown
---
name: my-skill
description: Описание когда активировать этот навык
---

# Инструкции

Когда пользователь просит X, делай Y...
```

### Встроенные навыки

| Навык | Описание |
|---|---|
| claude-api | Разработка с Anthropic SDK |
| feature-dev | 7-фазная разработка фич |
| frontend-design | UI без generic AI-эстетики |
| mcp-builder | Создание MCP-серверов |
| web-artifacts-builder | Сложные HTML-артефакты (React, Tailwind) |

### Добавление своих навыков

```bash
mkdir -p ~/workspace/.claude/skills/my-skill
cat > ~/workspace/.claude/skills/my-skill/SKILL.md << 'EOF'
---
name: my-skill
description: Когда пользователь спрашивает про деплой, используй этот гайд
---

# Деплой

1. Всегда деплоить на staging сначала
2. Проверить логи после деплоя
3. Уведомить в #deploys канал
EOF
```

Навык активируется со следующего сообщения — перезапуск не нужен.

## Расписания (Schedules)

Файл `~/.jarvis/schedules.json`:

```json
[
  {
    "id": "morning-report",
    "name": "Утренний отчёт",
    "type": "daily",
    "hour": 9,
    "minute": 0,
    "prompt": "Проверь статус проектов и напиши краткую сводку",
    "enabled": true
  },
  {
    "id": "weekly-review",
    "name": "Еженедельный обзор",
    "type": "weekly",
    "hour": 10,
    "weekdays": [1],
    "prompt": "Что сделали за неделю? git log за 7 дней по всем проектам",
    "enabled": true
  },
  {
    "id": "one-time-task",
    "name": "Напоминание",
    "type": "once",
    "at": "2026-05-01T12:00:00.000Z",
    "prompt": "Напомни про дедлайн по проекту X",
    "enabled": true
  }
]
```

Типы:
- **daily** — каждый день в hour:minute
- **weekly** — в определённые дни (weekdays: 1=Пн, 7=Вс)
- **once** — один раз в указанное время (at: ISO timestamp)

## Переменные окружения

Хранятся в `~/.jarvis/.env` (chmod 600). Два типа:

### Системные (управляются через /settings)

| Переменная | Обязательна | Описание |
|---|---|---|
| ENGINE | Нет | Движок: claude / codex (дефолт: claude) |
| BOT_TOKEN | Да | Telegram Bot Token от @BotFather |
| ANTHROPIC_API_KEY | Да* | Ключ Anthropic (* для engine=claude, вариант API) |
| CLAUDE_CODE_OAUTH_TOKEN | Да* | OAuth-токен подписки Claude Max (* альтернатива API-ключу) |
| OPENAI_API_KEY | Да* | Ключ OpenAI (* для engine=codex) |
| AGENT_NAME | Нет | Имя агента (дефолт: Джарвис) |
| ADMIN_ID | Нет | Telegram ID владельца |

Системные переменные скрыты из менеджера переменных — ими управляют кнопки Модель, Подключение, Дополнительно.

### Пользовательские (управляются через /settings → 🔑 Переменные окружения)

| Переменная | Описание |
|---|---|
| DEEPGRAM_API_KEY | Транскрипция голосовых |
| GITHUB_TOKEN | GitHub API (приватные репо, расширенные данные) |
| SERPER_API_KEY | Веб-поиск |
| TRUST_LEVEL | Ручной trust level: 0, 1, 2 |
| ... | Любые кастомные переменные |

Добавляются через inline UI: /settings → 🔑 Переменные окружения → ➕ Добавить.
Значения замаскированы (видны только последние 4 символа). Агент использует их через `$ИМЯ` в shell, не видит значений.
