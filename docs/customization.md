# Кастомизация

## Личность агента (SOUL.md)

Файл `~/workspace/SOUL.md` определяет КТО твой агент.

### Основные параметры

```markdown
# Личность

Имя: Фрайдей        # Имя агента
Стиль: формальный    # неформальный / формальный / нейтральный
Тон: деловой         # дружеский / деловой / академический
Язык: русский        # язык общения по умолчанию
```

### Конституция (приоритеты)

Пять принципов от P0 (высший) до P4 (низший):
- **P0 Agency** — агент как партнёр, не инструмент
- **P1 No Irreversible Harm** — вето на необратимые действия
- **P2 Honesty** — честность, без галлюцинаций
- **P3 Operator First** — приоритет целей владельца
- **P4 Action > Report** — делать, а не описывать

Можно убрать P0 если хочешь послушного исполнителя, или усилить если хочешь равного партнёра.

### Drift Detector

Список антипаттернов, которые агент сам отслеживает. Добавляй свои:

```markdown
## Drift Detector
1. Слишком длинные ответы → сокращай до 2-3 абзацев
2. Переключился на английский → вернись на русский
3. Забыл про дедлайн → проверь schedules.json
```

### Запрещённые паттерны

Фразы и эмодзи которые агент не должен использовать. Настрой под себя.

## Правила (CLAUDE.md)

Файл `~/workspace/CLAUDE.md` определяет КАК агент работает.

### Уровни автономности

Три уровня — настрой под свой комфорт:

**Максимальная автономия** (агент делает всё сам):
```markdown
GREEN: всё кроме удаления и деплоя
YELLOW: git push, npm publish
RED: rm -rf, DROP TABLE
```

**Минимальная автономия** (всё через подтверждение):
```markdown
GREEN: только чтение
YELLOW: создание файлов
RED: правка файлов, git, shell, деплой
```

### Память

Система памяти — три уровня:
- `~/workspace/MEMORY.md` — долгосрочная (факты, предпочтения, проекты)
- `~/workspace/memory/YYYY-MM-DD.md` — дневник (хронология, задачи дня)
- `~/workspace/knowledge/` — база знаний (архитектура, решения, конфиги)

Агент сам решает, что сохранять. Память эволюционирует: устаревшее архивируется,
важное укрепляется, дубли убираются.

### Формат ответов

По умолчанию — HTML для Telegram. Если используешь другой клиент, измени:
```markdown
## Формат ответа
- Markdown (для Discord/Slack)
- Обычный текст (для SMS/WhatsApp)
```

## Безопасность (hooks.js)

### Код-гейты

В отличие от промптов, хуки — это код. 100% гарантия блокировки.

Встроенные правила (активны по умолчанию):

**Деструктивные команды** — блокируются до подтверждения:
```
rm -rf, rmdir, sudo rm
DROP TABLE, TRUNCATE, DELETE FROM
git push --force, reset --hard, branch -D, clean -f
kill -9, pkill, shutdown, reboot
```

**Секреты в ответах** — маскируются автоматически:
```
sk-ant-api...  → sk-ant-***
ghp_xxxx...    → ghp_***
Bearer eyJ...  → Bearer ***
```

### Пользовательские хуки

Добавь свои правила в `~/.jarvis/hooks.json`:

```json
{
  "block": [
    { "pattern": "npm publish", "reason": "Публикация пакетов запрещена" },
    { "pattern": "docker rm", "reason": "Удаление контейнеров требует подтверждения" }
  ],
  "mask": [
    { "pattern": "PRIVATE_KEY_\\w+", "replace": "PRIVATE_KEY_***" }
  ]
}
```

## Уровень доверия (trust.js)

Агент «взрослеет» с опытом. В начале переспрашивает всё, через месяц — только критичное.

### Уровни

| Уровень | Условие | GREEN | YELLOW | RED |
|---|---|---|---|---|
| 0 — новичок | 0-10 сессий | auto | confirm | confirm |
| 1 — знакомый | 11-50 сессий | auto | auto+notify | confirm |
| 2 — доверенный | 51+ сессий | auto | auto | confirm (только критичные) |

### Ручная настройка

В `.env`:
```bash
TRUST_LEVEL=2  # Сразу максимальная автономия
```

Или не указывай — система сама определит по количеству сессий.

Файл `~/.jarvis/trust.json`:
```json
{
  "sessions": 42,
  "level": 1,
  "firstSeen": "2026-04-15",
  "overridden": false
}
```

## Навыки (Skills)

Навыки — markdown-файлы которые CLI-агент загружает автоматически.

### Структура навыка

```
~/workspace/.claude/skills/<name>/SKILL.md
```

Минимальный навык:
```markdown
---
name: my-skill
description: Описание когда активировать этот навык
---

# Инструкции

Когда пользователь просит X, делай Y...
```

### Встроенные навыки

| Навык | Описание |
|---|---|
| claude-api | Разработка с Anthropic SDK |
| feature-dev | 7-фазная разработка фич |
| frontend-design | UI без generic AI-эстетики |
| mcp-builder | Создание MCP-серверов |
| web-artifacts-builder | Сложные HTML-артефакты (React, Tailwind) |

### Добавление своих навыков

```bash
mkdir -p ~/workspace/.claude/skills/my-skill
cat > ~/workspace/.claude/skills/my-skill/SKILL.md << 'EOF'
---
name: my-skill
description: Когда пользователь спрашивает про деплой, используй этот гайд
---

# Деплой

1. Всегда деплоить на staging сначала
2. Проверить логи после деплоя
3. Уведомить в #deploys канал
EOF
```

Навык активируется со следующего сообщения — перезапуск не нужен.

## Расписания (Schedules)

Файл `~/.jarvis/schedules.json`:

```json
[
  {
    "id": "morning-report",
    "name": "Утренний отчёт",
    "type": "daily",
    "hour": 9,
    "minute": 0,
    "prompt": "Проверь статус проектов и напиши краткую сводку",
    "enabled": true
  },
  {
    "id": "weekly-review",
    "name": "Еженедельный обзор",
    "type": "weekly",
    "hour": 10,
    "weekdays": [1],
    "prompt": "Что сделали за неделю? git log за 7 дней по всем проектам",
    "enabled": true
  },
  {
    "id": "one-time-task",
    "name": "Напоминание",
    "type": "once",
    "at": "2026-05-01T12:00:00.000Z",
    "prompt": "Напомни про дедлайн по проекту X",
    "enabled": true
  }
]
```

Типы:
- **daily** — каждый день в hour:minute
- **weekly** — в определённые дни (weekdays: 1=Пн, 7=Вс)
- **once** — один раз в указанное время (at: ISO timestamp)

## Переменные окружения

В `~/.jarvis/.env`:

| Переменная | Обязательна | Описание |
|---|---|---|
| ENGINE | Нет | Движок: claude / codex (дефолт: claude) |
| BOT_TOKEN | Да | Telegram Bot Token от @BotFather |
| ANTHROPIC_API_KEY | Да* | Ключ Anthropic (* для engine=claude, вариант API) |
| CLAUDE_CODE_OAUTH_TOKEN | Да* | OAuth-токен подписки Claude Max (* альтернатива API-ключу) |
| OPENAI_API_KEY | Да* | Ключ OpenAI (* для engine=codex) |
| AGENT_NAME | Нет | Имя агента (дефолт: Джарвис) |
| ADMIN_ID | Нет | Telegram ID владельца |
| TRUST_LEVEL | Нет | Ручной trust level: 0, 1, 2 |
| DEEPGRAM_API_KEY | Нет | Для транскрипции голосовых |
| GITHUB_TOKEN | Нет | Для работы с GitHub |
| SERPER_API_KEY | Нет | Для веб-поиска |

Любые дополнительные переменные из `.env` будут доступны CLI-агенту через shell.
