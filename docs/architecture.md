# Архитектура

## Обзор

```
┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Telegram   │────▶│   bot.js         │────▶│  AI Engine      │
│   (grammy)   │◀────│   (Node.js)      │◀────│  CLI (stream)   │
└──────────────┘     └──────────────────┘     └─────────────────┘
                            │                        │
                     ┌──────┴──────┐          ┌──────┴──────┐
                     │  engine.js  │          │  workspace/  │
                     │  state.js   │          │  SOUL.md     │
                     │  media.js   │          │  CLAUDE.md   │
                     │  hooks.js   │          │  MEMORY.md   │
                     │  trust.js   │          │  skills/     │
                     │  scheduler  │          │  knowledge/  │
                     │  config     │          │  monitor     │
                     └─────────────┘          └─────────────┘
```

## Слои системы

Шесть слоёв:

| # | Слой | Что делает | Модуль |
|---|---|---|---|
| 1 | Обвязка | Telegram ↔ CLI, медиа, форматирование | bot.js, engine.js, media.js |
| 2 | Состояние | Персистентный стейт: сессии, расходы, режим | state.js |
| 3 | Безопасность | Код-гейты, маскировка, allowed/disallowed tools | hooks.js |
| 4 | Контроль | Permission modes (auto/control/plan), cost limits | state.js, engine.js |
| 5 | Оркестрация | Расписания, мониторинг, проактивные сканы | scheduler.js, monitor.js |
| 6 | Память | MEMORY.md, daily notes, knowledge/ | (внутри CLI-агента) |

Уровни 1-4 — детерминистические (код). Уровни 5-6 — вероятностные (модель).
Для необратимых операций решение принимает код (hooks + state), не модель.

## Поддерживаемые движки

| Движок | CLI | Стоимость | Качество |
|---|---|---|---|
| Claude Code | `claude` | $100/мес (Max) или API | Лучшее |
| OpenAI Codex | `codex` | $20/мес (Plus) или API | Отличное |

Оба движка работают с одними промптами, навыками и памятью.
Выбор: `ENGINE=claude|codex` в `.env`. Claude поддерживает два типа авторизации: API-ключ (`ANTHROPIC_API_KEY`) и OAuth-токен подписки (`CLAUDE_CODE_OAUTH_TOKEN`).

## Поток сообщения

```
Telegram → bot.js → [media.js] → engine.js → CLI (stream-json)
                                                       │
                                    state.js ← cost ←──┤
                                    hooks.js ← parse ←──┘
                                         │
                                   [блок / пропуск]
                                         │
                                  bot.js → Telegram
```

1. **Входящее** → Telegram update → grammy → admin check
2. **Батчинг** → сообщения за 500мс окно собираются в один запрос. Медиа (голос, фото, документы) скачиваются параллельно через `Promise.allSettled`
3. **Предобработка** → определение типа (текст/голос/фото/документ)
   - Голос: скачивание → Deepgram транскрипция → текст
   - Фото/документ: скачивание → директивный промпт с путём к файлу
4. **Очередь** → если сессия busy → «⏳ в очереди», добавить в messageQueue → обработка после
5. **Прогресс** → мгновенная отправка «🤔 Мозгую...» ДО запуска CLI (без задержки)
6. **State** → загрузка стейта: permissionMode, sessionId, costPaused
7. **Engine** → spawn CLI-процесса с полным набором аргументов:
   - Claude: `claude -p <prompt> --output-format stream-json --max-turns 25 --permission-mode bypassPermissions --allowedTools <tools> --disallowedTools <blocked>`
   - Codex: `codex --full-auto --quiet <prompt>`
8. **Stream parse (real-time)** → построчный парсинг JSON-стрима:
   - thinking → «🤔 Мозгую... Nс»
   - tool_use → «Читаю файл 🌚 Nс», «Редактирую 🌚 Nс» и т.д.
   - Прогресс-сообщение обновляется в Telegram (editMessageText)
9. **Error handling** → ошибки CLI (is_error) перехватываются, raw JSON не утекает в чат. Протухшие сессии (No conversation found) — авто-ретрай без --resume
10. **Cost tracking** → запись расхода в state.json, проверка лимита
11. **Hooks (пост)** → проверка ответа на секреты, деструктивные команды
12. **Постобработка** → удаление прогресс-сообщения, парсинг медиа-маркеров `[ФОТО:]`, `[ФАЙЛ:]`
13. **Футер** → `<blockquote>⏱ 35s · $0</blockquote>` — подписка $0, API key реальная стоимость — в составе последнего сообщения
14. **Исходящее** → медиа + текст + футер → Telegram. При лимите turns (20+ из 25) — инлайн-кнопки (✔ Продолжай / ✖ Стоп / 💬 Комментарий) → processQueue()

## Модули

### config.js
Загрузка `.env` из `~/.jarvis/.env`, определение движка и API-ключа, валидация обязательных параметров.

### state.js
Персистентный стейт в `~/.jarvis/state.json`. Атомарная запись (tmp + rename).

Хранит:
- `permissionMode` — auto / control / plan (дефолт: auto)
- `sessionId` — ID текущей CLI-сессии (для --resume)
- `costHistory` — расходы по дням { "2026-04-17": 1.23 } (30 дней)
- `dailySpendLimit` — лимит в USD (дефолт: $50)
- `costPaused` — автопауза при достижении лимита (авто-сброс при наступлении нового дня)
- `lastCostAlert` — дата последнего предупреждения (80%)
- `authMode` — subscription / api-key (автодетект)
- `activeProject` — текущий проект
- `timezone` — часовой пояс владельца

### engine.js
Абстракция над CLI-агентами (Claude Code, Codex). Единый интерфейс `send(prompt)` → `onDone(response, { cost, elapsed, authMode })`.

Построчный парсинг stream-json в реальном времени:
- `onProgress({ event, label, elapsed })` — live-статусы tool_use и thinking
- Маппинг tool names → человекочитаемые: Read→«Читаю файл», Edit→«Редактирую», Bash→«Выполняю команду», WebSearch→«Ищу в интернете», и т.д.
- Финальный `parseStreamJson()` — извлечение текста, cost, sessionId, numTurns (из `num_turns` в result или подсчётом assistant-сообщений)

CLI-аргументы Claude:
- `--output-format stream-json` — структурированный стрим (cost, sessionId, tool calls)
- `--max-turns 25` — ограничение итераций
- `--permission-mode bypassPermissions` — CLI не запрашивает подтверждений, безопасность обеспечена code-level guards (--disallowedTools, hooks.js, admin check)
- `--allowedTools <list>` — Bash, WebSearch, WebFetch + MCP-серверы
- `--disallowedTools <patterns>` — rm -rf, sudo, kill, pkill, shutdown, reboot, mkfs, dd
- `--resume <sessionId>` — восстановление сессии из state.json. При протухшей сессии — автоматический retry без --resume

Обработка ошибок CLI:
- `is_error` / `errors` из stream-json → человекочитаемое сообщение
- Протухшая сессия (No conversation found) → сброс sessionId → повторная отправка (один раз)
- Транзиентные ошибки (сеть, 500, rate limit) → авто-ретрай через 3с (один раз)
- Watchdog: если busy > 6 мин без завершения → kill + уведомление пользователю
- Raw JSON никогда не утекает в чат

Динамическая инъекция env-переменных: `getUserEnvKeys()` читает `~/.jarvis/.env`, фильтрует системные и пустые, инжектит имена в контекст-префикс каждого запроса. Агент знает какие переменные доступны.

Управление сессиями, таймаутами (5 мин), авто-очистка idle сессий (10 мин). CWD = `resolveProjectDir()` — переключается вместе с текущим проектом.

### menu.js
Главное меню с 8 inline-кнопками (Новая сессия, Проекты, Сессии, Навыки, Сервер, MCP, Настройки, Мониторинг). Показывается после `/start` для онбордённых пользователей.

### projects.js
Управление проектами: список из `~/workspace`, подпапок `~/workspace/*` (кроме служебных: memory, knowledge, .claude, .git) и `~/projects/*`. Системные проекты (jarvis-bootstrap, jarvis-installer, helper-aishnik) скрыты из списка. Пагинация (9 на страницу), переключение текущего проекта, сохранение в `~/.jarvis/project.json`. При переключении — сброс CLI-сессии (killSession + setSessionId(null)). «Новый проект» создаёт в `~/projects/`. `resolveProjectDir()` — резолвит путь текущего проекта для CWD движка.

Callback data кнопок: `projects:s:<index>` (числовой индекс в массиве проектов) вместо полного имени — лимит Telegram 64 байта, кириллица = 2 байта/символ в UTF-8. Обратная совместимость со старым форматом `projects:switch:<name>` (кешированные кнопки).

UX кнопок: `displayName()` убирает `~/workspace/` префикс — на кнопках только имя папки. Layout 2 в ряд (больше места для имён), truncate до 20 символов. При переключении — явное подтверждение отдельным сообщением (как `/project` команда), не только toast.

### media.js
Скачивание файлов из Telegram API (с проверкой HTTP-статуса, timeout 30с), транскрипция голосовых через Deepgram Nova-2 (русский),
парсинг медиа-маркеров `[ФОТО/ФАЙЛ/ВИДЕО/АУДИО/ГОЛОС/GIF:]` в ответах, отправка медиа.
Авто-очистка старых медиафайлов (>24ч) при старте и каждый час.

### hooks.js
Код-гейты безопасности + md→html конвертация. Детерминистический слой — модель не участвует в решении.

Четыре типа обработки:
- **pre-command** — `checkCommand()` блокирует деструктивные операции (`rm -rf`, `DROP TABLE`, `git push --force`, `kill`, `shutdown`). Не подключён к потоку промптов (ложные срабатывания на естественном языке) — безопасность обеспечена через `--disallowedTools` в engine.js.
- **post-response** — `maskSecrets()` сканирует ответ агента. Маскирует секреты (API-ключи, токены, пароли по паттернам).
- **input-detect** — `detectSensitiveInput()` сканирует входящие сообщения пользователя на наличие секретов (11 типов: Anthropic, OpenAI, GitHub, Slack, AWS, JWT, PEM). Для ключей движков — автосохранение + предупреждение. Для остальных — предупреждение + совет использовать менеджер переменных.
- **md→html** — `mdToHtml()` конвертирует markdown в Telegram HTML:
  1. Извлечение code-блоков и inline-code (с HTML entity escaping внутри)
  2. Сохранение валидных Telegram тегов (`b|i|u|s|code|pre|a|blockquote|tg-spoiler|tg-emoji`)
  3. Escaping HTML-сущностей (`&`, `<`, `>`) в plain text
  4. Конвертация: `**bold**`→`<b>`, `*italic*`→`<i>`, `[text](url)`→`<a href>`, `### heading`→`<b>`, `- item`→`[x] item`
  5. `autoCodeWrap()` — автооборачивание @mentions и имён файлов (.js, .py, .md и др.) в `<code>`, не трогая содержимое `<code>`/`<pre>`

Конфиг хуков: `~/.jarvis/hooks.json` (опционально, дефолтные правила встроены).

### trust.js
Динамический уровень доверия (наследие, совместим с permission modes).
Растёт с количеством сессий. Счётчик в `~/.jarvis/trust.json`.

> **Permission modes** (из state.js) — основной механизм контроля:
>
> CLI всегда запускается с `--permission-mode bypassPermissions` — безопасность обеспечена на уровне кода (--disallowedTools, hooks.js, admin check), а не через интерактивные подтверждения CLI.
>
> Permission modes в UI (auto/control/plan) — наследие, сохранены для будущей гранулярности.
> При смене режима сессия сбрасывается (sessionId = null).
> Режим хранится в state.json, UI в settings.js.

### monitor.js
Мониторинг источников: RSS/Atom, GitHub releases, YouTube. Все типы работают без API-ключей — через публичные RSS/Atom фиды. Пользователь добавляет источники через inline-UI.

Конфиг: `~/.jarvis/monitor.json` — список источников, digestHour, enabled. Seen-дедупликация: `~/.jarvis/monitor/seen.json` (до 200 записей/источник).

Fetchers: `fetchRSS()` — парсинг XML без зависимостей (regex на `<item>` / `<entry>`, извлечение description/summary); `fetchGitHub()` — делегирует в `fetchRSS()` через `releases.atom` (без токена); `fetchYouTube()` — делегирует в `fetchRSS()` через YouTube RSS feed (без API-ключа). Все с `AbortSignal.timeout(15000)`.

YouTube: автоконвертация ссылок (`m.youtube.com`, `youtube.com/@handle`, `youtube.com/channel/UCxxx`) в RSS-фид. `resolveYoutubeChannelId()` — fetch страницы канала → извлечение UC-идентификатора из HTML.

Саммари: `summarizeItems()` — если у пользователя есть API-ключ движка (ANTHROPIC_API_KEY → Haiku, OPENAI_API_KEY → GPT-4o-mini), новые записи получают краткое саммари (1-2 предложения). Батч-запрос на все элементы разом. При подписке (OAuth-only) — без саммари.

Inline-UI: `buildMonitorKeyboard()` — список источников с toggle enabled/disabled, добавление (выбор типа → ввод URL/ссылки), удаление, «Проверить сейчас». `pendingAdd` — FSM-состояние для многошагового добавления. `handleMonitorCallback()` — роутинг `mon:*` callbacks. `handleMonitorInput()` — обработка текстового ввода при добавлении (async, резолвит @handle).

Интеграция с scheduler: проверка каждые 30 мин, дайджест в `digestHour` (из конфига) или при 10+ новых.

### scheduler.js
Расписания из `~/.jarvis/schedules.json`. Типы: daily, weekly, once.
Проверка каждые 60 сек, защита от дублей (мин. 1 час между запусками).
Timezone-aware: `getNowInTz()` использует `Intl.DateTimeFormat.formatToParts()` для точного определения часа, минуты и дня недели в часовом поясе пользователя (из `state.timezone`).
Ответы проходят через `processResponse()` (маскировка секретов, md→html) и `parseMediaMarkers()`. Пропускает выполнение если сессия пользователя busy.

### settings.js
Меню настроек: статус-карта (подключение, режим, модель, часовой пояс, лимит, расходы), 9 inline-кнопок. Подключение движков с пошаговой инструкцией. Автодетект OAuth-токенов и API-ключей. Очистка whitespace из токенов (защита от обрезки Telegram). Менеджер переменных окружения: список с маскировкой значений, добавление/удаление через inline-кнопки, системные переменные скрыты, опасные переменные заблокированы (LD_PRELOAD, NODE_OPTIONS, PATH и др.).

### bot.js
Telegram бот (grammy + auto-retry): команды `/start`, `/project`, `/monitor`, `/settings`, `/status`, 18 команд через `setMyCommands`. `bot.catch()` — graceful error handling. Роутинг callback-кнопок (menu:*, settings:*, projects:*, mon:*, action:*) с admin check на всех обработчиках. Автодетект токенов в сообщениях и файлах. Обработчики текста, голоса, фото, документов, видео, стикеров, аудио. Перехват текстового ввода при `pendingAdd` (добавление источника мониторинга).

Батчинг: входящие сообщения за 500мс окно собираются в один запрос. Скачивание медиа (фото, голос, документы) запускается параллельно через `Promise.allSettled`. Несколько сообщений объединяются в один промпт для CLI.

Мгновенный прогресс: «🤔 Мозгую...» отправляется ДО запуска CLI (без задержки 2-3с). Далее одно обновляемое сообщение показывает текущее действие агента (Читаю файл, Редактирую...) + таймер. Удаляется после получения ответа.

Футер: `<blockquote>⏱ 35s · $0</blockquote>` в составе последнего сообщения ответа. При подписке (subscription) — всегда `$0` (фиксированная плата, теоретический cost из CLI не записывается). При API key — реальная стоимость из stream-json.

splitMessage: разбивка по 4000 символов с `closeOpenTags()` — незакрытые HTML-теги (`<b>`, `<i>`, `<code>` и т.д.) корректно закрываются на границе чанка и переоткрываются в начале следующего.

Голосовые транскрипции: HTML-экранируются перед отображением (защита от инъекций через голосовые).

Инлайн-кнопки: появляются только когда агент упёрся в лимит turns (20+ из 25 — задача не завершена). Три кнопки (✔ Продолжай / ✖ Стоп / 💬 Комментарий, layout 2+1). Количество turns определяется из `num_turns` в result stream-json или подсчётом assistant-сообщений в стриме. «Продолжай» отправляет «Продолжай.» в движок, «Стоп» убивает текущую сессию, «Комментарий» ждёт следующего сообщения.

Очередь сообщений: если агент занят — новый запрос ставится в очередь («⏳ в очереди»), обрабатывается автоматически после завершения текущей задачи.

Typing-индикатор: отправляется ПОСЛЕ прогресс-сообщения (reply сбрасывает typing), интервал 3с, refresh после каждого editMessageText. Разбивка длинных ответов (4000 символов). HTML fallback: при ошибке парсинга HTML теги стрипаются, не показываются сырыми.

Graceful shutdown: при SIGTERM/SIGINT — уведомление пользователю («Перезапускаюсь...»), killAllSessions(), bot.stop(), exit.

## Cost Tracking

Расходы извлекаются из stream-json вывода CLI и накапливаются в state.json.
**Только для API key** — при подписке (subscription) cost из CLI теоретический и не записывается.

```
CLI stream-json → парсинг cost_usd → [subscription? skip] → state.costHistory[date] += cost
                                            │
                                    ┌───────┴────────┐
                                    │ dailySpendLimit │
                                    │ (дефолт $50)    │
                                    └───────┬────────┘
                                            │
                               80% → предупреждение
                              100% → costPaused = true → блок запросов
```

- История хранится 30 дней, старые записи автоочищаются
- Лимит настраивается через /settings → Лимит расходов
- При паузе — сообщение пользователю, запросы не проходят. Авто-сброс паузы при наступлении нового дня (расход обнулился)
- /cost показывает расход за сегодня и историю

## Авторизация

Два режима, автодетект:

| Режим | Переменная | Детект | Особенности |
|---|---|---|---|
| subscription | `CLAUDE_CODE_OAUTH_TOKEN` | Токен `sk-ant-oat` / `sk-ant-ort` | Персистентные сессии через sessionId |
| api-key | `ANTHROPIC_API_KEY` | Обычный API-ключ | Stateless, каждый запрос отдельно |

Автодетект: если `ANTHROPIC_API_KEY` содержит subscription-токен (`sk-ant-oat*`), он автоматически переносится в `CLAUDE_CODE_OAUTH_TOKEN`. authMode сохраняется в state.json.

## Идентичность агента (TELOS)

Пять файлов определяют агента:

| Файл | Что определяет |
|---|---|
| `SOUL.md` | КТО — имя, стиль, тон, конституция P0-P4 |
| `CLAUDE.md` | КАК — правила, автономность, безопасность |
| `MEMORY.md` | ЧТО ЗНАЕТ — факты, проекты, предпочтения |
| `memory/*.md` | ЧТО БЫЛО — дневник, хронология |
| `knowledge/` | ЧТО ПОНИМАЕТ — архитектура, решения, конфиги |

При смене движка — идентичность сохраняется. Накопленные знания не привязаны к провайдеру.

## Безопасность

### Уровни (код + промпт)

| Уровень | Механизм | Гарантия |
|---|---|---|
| engine.js (код) | `--disallowedTools` — блокировка rm, sudo, kill, pkill, shutdown, reboot, mkfs, dd на уровне CLI | 100% — код всегда исполняется |
| engine.js (код) | `bypassPermissions` + code-level guards — CLI не запрашивает подтверждений | 100% — код всегда исполняется |
| hooks.js (код) | Блокировка деструктивных команд, маскировка секретов в ответах | 100% — код всегда исполняется |

Принцип: всё критичное — в коде (hooks + settings). Промпт — второй эшелон.

### Защита данных
- `.env chmod 600` — ключи доступны только владельцу
- `ADMIN_ID` — ограничение доступа одним пользователем (проверяется на ВСЕХ обработчиках)
- hooks.js post-response — автоматическая маскировка секретов в ответах
- hooks.js input-detect — предупреждение при отправке секретов в чат
- settings.js — менеджер переменных через inline-кнопки, значения замаскированы, опасные имена заблокированы
- onboarding.js — HTML-экранирование имени владельца (защита от инъекций)
- Агент использует переменные через `$ИМЯ` в shell, не видит значений в своём контексте
