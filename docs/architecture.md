# Архитектура

## Обзор

```
┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Telegram   │────▶│   bot.js         │────▶│  AI Engine      │
│   (grammy)   │◀────│   (Node.js)      │◀────│  CLI            │
└──────────────┘     └──────────────────┘     └─────────────────┘
                            │                        │
                     ┌──────┴──────┐          ┌──────┴──────┐
                     │  engine.js  │          │  workspace/  │
                     │  media.js   │          │  SOUL.md     │
                     │  hooks.js   │          │  CLAUDE.md   │
                     │  trust.js   │          │  MEMORY.md   │
                     │  scheduler  │          │  skills/     │
                     │  config     │          │  knowledge/  │
                     └─────────────┘          └─────────────┘
```

## Слои системы

Четыре слоя, каждый решает свою задачу:

| # | Слой | Что делает | Модуль |
|---|---|---|---|
| 1 | Обвязка | Telegram ↔ CLI, медиа, форматирование | bot.js, engine.js, media.js |
| 2 | Безопасность | Код-гейты для деструктивных операций | hooks.js |
| 3 | Оркестрация | Расписания, проактивные сканы | scheduler.js |
| 4 | Память | MEMORY.md, daily notes, knowledge/ | (внутри CLI-агента) |

Важно: уровни 1-2 — детерминистические (код). Уровень 3-4 — вероятностные (модель).
Для необратимых операций решение принимает код (hooks), не модель.

## Поддерживаемые движки

| Движок | CLI | Стоимость | Качество |
|---|---|---|---|
| Claude Code | `claude` | $100/мес (Max) или API | Лучшее |
| OpenAI Codex | `codex` | $20/мес (Plus) или API | Отличное |
| Gemini CLI | `gemini` | Бесплатно (Google) | Хорошее |

Все три движка работают с одними промптами, навыками и памятью.
Выбор: `ENGINE=claude|codex|gemini` в `.env`.

## Поток сообщения

```
Telegram → bot.js → [media.js] → engine.js → [hooks.js] → CLI → ответ
                                                                   │
                                              hooks.js ← парсинг ←─┘
                                                │
                                          [блок / пропуск]
                                                │
                                         bot.js → Telegram
```

1. **Входящее** → Telegram update → grammy
2. **Предобработка** → определение типа (текст/голос/фото/документ)
   - Голос: скачивание → Deepgram транскрипция → текст
   - Фото/документ: скачивание → путь в промпт
3. **Engine** → spawn CLI-процесса:
   - Claude: `claude --print --output-format text <prompt>`
   - Codex: `codex --full-auto --quiet <prompt>`
   - Gemini: `gemini --noinput <prompt>`
4. **Hooks (пост)** → проверка ответа на секреты, деструктивные команды
5. **Постобработка** → парсинг медиа-маркеров `[ФОТО:]`, `[ФАЙЛ:]`
6. **Исходящее** → медиа + текст → Telegram

## Модули

### config.js
Загрузка `.env` из `~/.jarvis/.env`, определение движка и API-ключа, валидация обязательных параметров.

### engine.js
Абстракция над тремя CLI-агентами. Единый интерфейс `send(prompt)` → `onDone(response)`.
Управление сессиями, таймаутами (5 мин), авто-очистка idle сессий (10 мин).

### media.js
Скачивание файлов из Telegram API, транскрипция голосовых через Deepgram Nova-2 (русский),
парсинг медиа-маркеров `[ФОТО/ФАЙЛ/ВИДЕО/АУДИО/ГОЛОС/GIF:]` в ответах, отправка медиа.

### hooks.js
Код-гейты безопасности. Детерминистический слой — модель не участвует в решении.

Два типа хуков:
- **pre-command** — перехватывает команды перед выполнением CLI. Блокирует деструктивные
  операции (`rm -rf`, `DROP TABLE`, `git push --force`, `kill`, `shutdown`).
- **post-response** — сканирует ответ агента перед отправкой в Telegram. Маскирует секреты
  (API-ключи, токены, пароли по паттернам).

Конфиг хуков: `~/.jarvis/hooks.json` (опционально, дефолтные правила встроены).

### trust.js
Динамический уровень доверия. Растёт с количеством сессий:

| Уровень | Сессий | Поведение |
|---|---|---|
| 0 — новичок | 0-10 | Подтверждение на YELLOW + RED |
| 1 — знакомый | 11-50 | Подтверждение только на RED |
| 2 — доверенный | 51+ | Подтверждение только на критические RED |

Счётчик хранится в `~/.jarvis/trust.json`. Уровень можно задать вручную в `.env` (`TRUST_LEVEL=2`).

### scheduler.js
Расписания из `~/.jarvis/schedules.json`. Типы: daily, weekly, once.
Проверка каждые 60 сек, защита от дублей (мин. 1 час между запусками).

### bot.js
Telegram бот (grammy + auto-retry): команды `/start`, `/reset`, `/status`,
обработчики текста, голоса, фото, документов.
Typing-индикатор, разбивка длинных ответов (4000 символов), graceful shutdown.

## Идентичность агента (TELOS)

Пять файлов определяют агента:

| Файл | Что определяет |
|---|---|
| `SOUL.md` | КТО — имя, стиль, тон, конституция P0-P4 |
| `CLAUDE.md` | КАК — правила, автономность, безопасность |
| `MEMORY.md` | ЧТО ЗНАЕТ — факты, проекты, предпочтения |
| `memory/*.md` | ЧТО БЫЛО — дневник, хронология |
| `knowledge/` | ЧТО ПОНИМАЕТ — архитектура, решения, конфиги |

При смене движка — идентичность сохраняется. Накопленные знания не привязаны к провайдеру.

## Безопасность

### Уровни (код + промпт)

| Уровень | Механизм | Гарантия |
|---|---|---|
| hooks.js (код) | Блокировка деструктивных команд, маскировка секретов | 100% — код всегда исполняется |
| CLAUDE.md (промпт) | GREEN/YELLOW/RED уровни автономности | ~70% — модель может нарушить |
| SOUL.md (промпт) | P1 No Irreversible Harm | ~70% — дублирует на уровне ценностей |

Принцип: всё критичное — в коде (hooks). Промпт — второй эшелон.

### Защита данных
- `.env chmod 600` — ключи доступны только владельцу
- `ADMIN_ID` — ограничение доступа одним пользователем
- hooks.js post-response — автоматическая маскировка секретов в ответах
- Агент никогда не видит значения ключей в своём контексте

## Эволюция агента

### Фаза 1: Персональный агент (текущая)
Один агент, один владелец. Инструменты, память, навыки.

### Фаза 2: Рост доверия
Динамический trust level. Агент начинает делать больше без спроса.
Автоматическое извлечение навыков из рабочих паттернов.

### Фаза 3: Проактивность
Агент сам сканирует проекты, замечает задачи, предлагает решения.
Утренние/вечерние сводки без запроса.

### Фаза 4: Команда агентов (roadmap)
Специализированные агенты наследуют контекст ядра.
Координация через человека (не автоматический оркестратор).

### Фаза 5: Межагентное взаимодействие (roadmap)
A2A/MCP протоколы. Агенты общаются с агентами других людей и сервисов.

## Структура на сервере

```
~/.jarvis/
├── .env              # ENGINE + ключи (chmod 600)
├── schedules.json    # Расписания
├── hooks.json        # Пользовательские хуки (опционально)
└── trust.json        # Счётчик сессий и trust level

~/workspace/
├── SOUL.md           # Идентичность
├── CLAUDE.md         # Правила
├── MEMORY.md         # Долгосрочная память
├── memory/           # Дневник (YYYY-MM-DD.md)
├── knowledge/        # База знаний
└── .claude/skills/   # Навыки (автозагрузка)

~/projects/           # Рабочие проекты
```
