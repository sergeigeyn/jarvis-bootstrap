# Архитектура

## Обзор

```
┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Telegram   │────▶│   bot.js         │────▶│  Claude Code    │
│   (grammy)   │◀────│   (Node.js)      │◀────│  CLI (--print)  │
└──────────────┘     └──────────────────┘     └─────────────────┘
                            │                        │
                     ┌──────┴──────┐          ┌──────┴──────┐
                     │  media.js   │          │  workspace/  │
                     │  scheduler  │          │  SOUL.md     │
                     │  config     │          │  CLAUDE.md   │
                     └─────────────┘          │  skills/     │
                                              │  memory/     │
                                              └─────────────┘
```

## Поток сообщения

1. **Входящее** → Telegram отправляет webhook/polling update в grammy
2. **Предобработка** → bot.js определяет тип (текст/голос/фото/документ)
   - Голос: скачивается → транскрибируется через Deepgram → текст
   - Фото: скачивается → путь передаётся Claude для чтения
   - Документ: скачивается → путь передаётся Claude
3. **Claude Code CLI** → spawn процесса `claude --print --output-format text`
   - Работает в контексте ~/workspace/ (читает SOUL.md, CLAUDE.md)
   - Полный доступ к файловой системе, shell, git, интернету
   - Таймаут 5 минут на ответ
4. **Постобработка** → парсинг медиа-маркеров в ответе
   - `[ФОТО: /path/to/file]` → отправка фото
   - `[ФАЙЛ: /path/to/file]` → отправка документа
   - Текст разбивается на чанки по 4000 символов
5. **Исходящее** → медиа + текст отправляются в Telegram

## Модули

### config.js (48 строк)
Загрузка конфигурации из ~/.jarvis/.env и переменных окружения.
Валидация обязательных параметров (BOT_TOKEN, ANTHROPIC_API_KEY).

### claude-session.js (113 строк)
Управление процессами Claude Code CLI:
- По одной сессии на chatId
- Защита от параллельных запросов (busy flag)
- Автоочистка неактивных сессий (10 мин таймаут)
- Graceful kill: SIGTERM → 5s → SIGKILL

### media.js (145 строк)
Три функции:
- **downloadFile()** — скачивание файлов из Telegram Bot API
- **transcribeVoice()** — отправка аудио в Deepgram Nova-2 API
- **parseMediaMarkers()** — парсинг маркеров [ФОТО:], [ФАЙЛ:] и т.д. в ответе Claude

Поддерживаемые маркеры: ФОТО/PHOTO, ФАЙЛ/FILE, СТИКЕР/STICKER, ВИДЕО/VIDEO, АУДИО/AUDIO, ГОЛОС/VOICE, GIF/ANIMATION.

### scheduler.js (104 строк)
Система расписаний:
- Хранение в ~/.jarvis/schedules.json
- Типы: daily (ежедневно), weekly (по дням недели), once (одноразовый)
- Проверка каждую минуту
- Защита от дублирования (минимум час между запусками)
- Выполнение через Claude Code CLI, результат → в Telegram админу

### bot.js (226 строк)
Главный файл:
- Инициализация grammy с auto-retry
- Проверка доступа (ADMIN_ID)
- Обработчики: /start, /reset, /status, текст, голос, фото, документы
- Разбивка длинных сообщений (splitMessage)
- Typing индикатор во время обработки
- Graceful shutdown на SIGINT/SIGTERM

## Безопасность

### Доступ
- ADMIN_ID в .env ограничивает бота одним пользователем
- Если ADMIN_ID не задан — бот доступен всем (не рекомендуется)

### Ключи
- .env файл: chmod 600, доступен только владельцу
- Claude Code получает ANTHROPIC_API_KEY через переменные окружения
- SOUL.md/CLAUDE.md содержат инструкцию не показывать ключи в чате

### Уровни автономности (в CLAUDE.md)
- **GREEN** — автоматически: чтение, поиск, заметки
- **YELLOW** — с уведомлением: правка файлов, git commit
- **RED** — только с подтверждением: удаление, push, деплой
