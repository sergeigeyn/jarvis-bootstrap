# Frontend Design Skill

## Core Purpose
Build distinctive, production-grade web interfaces. Reject generic "AI slop" aesthetics.

## Design Approach — Before Coding

Commit to a BOLD aesthetic direction:
- **Purpose**: What problem does this solve? Who uses it?
- **Tone**: Pick an extreme direction (brutally minimal, maximalist, retro-futuristic, luxury, playful, industrial, etc.)
- **Constraints**: Technical requirements
- **Differentiation**: What makes it UNFORGETTABLE?

Core principle: Choose a clear conceptual direction and execute it with precision.

## Implementation Requirements
Code must be: production-grade, visually striking, cohesive with clear aesthetic point-of-view, meticulously refined.

## Five Key Areas

### 1. Typography
Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts (Arial, Inter, Roboto). Pair a display font with a refined body font.

### 2. Color & Theme
Commit to cohesive palettes using CSS variables. Dominant colors with sharp accents > timid palettes.

### 3. Motion
CSS animations, scroll-triggering, hover states. Prioritize high-impact page load reveals over scattered micro-interactions.

### 4. Spatial Composition
Unexpected layouts, asymmetry, overlap, diagonal flow, grid-breaking elements.

### 5. Backgrounds & Visual Details
Gradient meshes, noise textures, geometric patterns, layered transparencies, decorative borders.

## What NOT to Do
- Avoid overused fonts (Inter, Roboto, Arial, system fonts)
- Avoid cliched color schemes (purple gradients on white)
- Avoid predictable, cookie-cutter designs
- Never converge on common choices across different projects
- Avoid excessive centered layouts and uniform rounded corners
